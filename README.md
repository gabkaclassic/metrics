# Metrics

## Профилирование проекта
1) **Соберите проект**
`make build MODULE=agent & make build MODULE=server`
2) **Запустите agent и server**
`go build/server <OPTIONS>`
и
`go build/agent <OPTIONS>`
3) **Запустите профилирование**
`make profile <OPTIONS>`
### Результаты улучшения с помощью анализа profile
При анализе профиля было выявлено достаточно большое количество аллокаций. После темы антипаттернов сразу вспомнились аллокации при передаче указателей на структуры/слайсы/мапы (т.к изначально этот момент казался контринтуитивным), так что было принято решение исправить этот косяк, который встречался по всему проекту. Повторное профилирование после изменений показало, что ресурсов на аллокацию было действительно затрачено меньше, что подтвердило информацию про аллокации при передаче указателей
